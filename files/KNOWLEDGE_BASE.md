# База знаний BCC — Полный анализ

## Общая информация
- **Количество файлов:** 21 (в подпапках `картa/`, `кн/`, `автокредит/`, `депозиты/`, `ипотека/`, `залог/`)
- **Формат:** MHTML-экспорт из Confluence (20 файлов) + Word 97 (1 файл — `общие условия.doc`)
- **Путь в проекте:** `backend/knowledge/`
- **Язык:** Русский (основной)
- **Суммарный объём:** ~600 000 символов чистого текста (после парсинга), ~354 чанка

---

## КРИТИЧНО: Как читаются файлы

Парсинг реализован в `backend/context.py` через **BeautifulSoup** (формат определяется
автоматически по сигнатуре файла):

- **MHTML** (20 файлов): извлекается HTML → BeautifulSoup:
  - реальные `<table>` → markdown-таблицы (строки/колонки сохраняются);
  - карточки Confluence `aura-card` → `**заголовок** — значение`;
  - фильтр base64-блобов встроенных картинок (иначе `общие условия.doc` раздувается до ~2.7M символов мусора).
- **Word 97** (`общие условия.doc`): конвертация через LibreOffice (`soffice --convert-to docx`) → `python-docx`.

> Старый подход на голом `html2text` расплющивал таблицы ставок/тарифов в плоский
> список чисел без привязки к продукту — поэтому заменён на BeautifulSoup.
