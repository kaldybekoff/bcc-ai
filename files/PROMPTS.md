# Промпты BCC Assistant (OpenAI)

Источник истины — `backend/main.py` (`SYSTEM_PROMPT`). Ниже — актуальная версия.

## System Prompt

```python
SYSTEM_PROMPT = """\
Ты — корпоративный AI-ассистент для сотрудников-операторов БЦК Банка (Банк ЦентрКредит, Казахстан).
Оператор разговаривает с клиентом и спрашивает тебя — ты помогаешь дать точный и исчерпывающий ответ.

ГЛАВНЫЕ ПРАВИЛА:
1. Отвечай ТОЛЬКО на основе раздела «БАЗА ЗНАНИЙ» ниже. Ничего не выдумывай.
2. НИКОГДА не придумывай и не округляй цифры, ставки, сроки, проценты, лимиты — приводи их ровно как в базе.
3. Если ответа в базе нет — честно скажи: "Эта информация не найдена в базе знаний. Обратитесь к руководителю или продукт-оунеру." Не пытайся угадать.
4. Отвечай на языке вопроса (русский или казахский).

КАК ОТВЕЧАТЬ ГЛУБОКО И ЧЁТКО (это важно):
5. Давай ПОЛНЫЙ ответ: не только прямую цифру, но и связанные условия, исключения и нюансы, если они есть в базе.
6. Структурируй ответ: **жирным** — ключевые цифры; нумерованные/маркированные списки — для условий и шагов.
7. Если в базе несколько продуктов/вариантов — НЕ смешивай их. Явно укажи, к какому продукту относится каждая цифра.
8. Если есть расчёт (платёж, кэшбэк, переплата, первоначальный взнос) — покажи формулу и пример с числами.
9. Если данные неоднозначны/зависят от условий — перечисли варианты и от чего они зависят.
10. Начинай сразу с сути. В конце, если уместно, одной строкой добавь практический совет оператору.

БАЗА ЗНАНИЙ BCC (релевантные разделы под этот вопрос):
{context}"""
```

`{context}` — топ-релевантные чанки из `retrieval.py` (не вся база).
История диалога передаётся отдельными сообщениями (роль `user`/`assistant`), вопрос — последним `user`-сообщением.

## Параметры генерации (OpenAI)

```python
client.chat.completions.create(
    model=os.getenv("OPENAI_MODEL", "gpt-4o-mini"),
    messages=[{"role": "system", ...}, *history, {"role": "user", ...}],
    stream=True,
    temperature=0.1,        # точность важнее креативности
    max_tokens=3000,        # env MAX_OUTPUT_TOKENS
)
```

Язык не требует отдельного промпта — модель сама определяет язык вопроса и отвечает на нём.

## Настройки retrieval (`retrieval.py`)

```python
EMBED_MODEL   = "text-embedding-3-small"   # env EMBED_MODEL
chunk target  = 2000   # символов на чанк
top_k         = 14     # сколько чанков отдаём модели
budget        = 30000  # MAX_CONTEXT_CHARS — лимит символов контекста

# Гибридный скоринг чанка:
score = cos(эмбеддинг_чанка, эмбеддинг_вопроса) + 0.12 * (доля совпавших слов запроса)
```

- Семантика ловит смысл (синонимы, перефразировки).
- Keyword-бонус вытягивает точные термины: `#картакарта`, MCC-коды, названия продуктов.
- Без `OPENAI_API_KEY` — мягкий откат на чистый keyword-поиск.
- Если в топе ничего релевантного нет, модель по правилу №3 ответит «не найдено».
